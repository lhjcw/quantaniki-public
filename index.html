IMPLEMENTED CAPABILITIES

不是概念列表，是已经接进工作流的能力。

以下能力均能在当前前端、接口或测试中找到对应实现。

01DIALOG → DSL

对话式策略构建

逐步补齐进场、出场、商品、周期、仓位与成本，并生成结构化策略。

02PARSER → IR

Pine 安全子集分析

针对 Pine v5/v6 的可安全执行子集，经 Parser、AST、IR 与语义审计标记不支持功能、重绘风险和撮合差异。

03COARSE → VALUE

参数价值研究

先选值得研究的参数，再执行受约束的粗扫、价值分析与后续精细验证。

04PORTFOLIO / SYMBOL

回测与分商品诊断

读取组合权益、交易明细、胜率、回撤、Sharpe、盈利因子和每个商品表现。

05OOS / WFO / STRESS

六类稳定性测试

覆盖样本外、Walk-forward、市场状态、成本滑点、参数邻域与随机扰动。

06REPORT → NEXT

带报告上下文的 Aniki

结合当前参数、回测和稳定性报告解释结果，继续调整策略或安排下一轮实验。

PINE COMPATIBILITY LAYER

不直接相信脚本；先证明它能安全进入研究流程。

QuantAniki 支持 Pine v5/v6 的可安全执行子集，不等同于完整 TradingView 兼容。源码先经过结构、语义、时间与资源检查；不支持的分支或不完整转换会 fail closed。

FAQ

开始之前，你可能想知道。

01不会写代码也能建立策略吗？＋

可以。Aniki 会通过对话追问商品、周期、进出场、仓位和成本，并把结果整理成策略条件。

02Pine Script 是直接执行吗？＋

不是。当前仅支持 Pine v5/v6 的安全子集，并先通过语义、来源与执行门禁；这不等同完整 TradingView 兼容。

03参数优化会不会只是暴力搜索？＋

产品先识别可调参数和研究价值，再生成受约束的粗扫计划，并检查执行绑定、交易数量与结果有效性。

04它会连接交易所自动下单吗？＋

不会。当前是本地策略研究工具，数据下载不等于实盘交易，也没有券商或交易所下单连接。
