IMPLEMENTED CAPABILITIES

不是概念列表，是已接進工作流程的能力。

以下能力均能在當前前端、介面或測試中找到對應實作。

01DIALOG → DSL

對話式策略構建

補齊進場、出場、商品、週期、倉位與成本，生成結構化策略。

02PARSER → IR

Pine 安全子集分析

針對 Pine v5/v6 的安全子集，經 Parser、AST、IR 與語義審計標記不支援功能、重繪與撮合差異。

03COARSE → VALUE

04PORTFOLIO / SYMBOL

05OOS / WFO / STRESS

六類穩定性測試

覆蓋樣本外、Walk-forward、市場狀態、成本滑點、參數鄰域與隨機擾動。

06REPORT → NEXT

PINE COMPATIBILITY LAYER

不直接相信腳本；先證明它能安全進入研究流程。

QuantAniki 支援 Pine v5/v6 的可安全執行子集，不等同完整 TradingView 相容。源碼先經結構、語義、時間與資源檢查；不支援的分支會 fail closed。

01

源碼覆蓋

保留來源與轉換節點，檢查 IR 能否追溯到源碼。

02

語義審計

分析 series、歷史引用、多週期、訂單與執行語義。

03

風險報告

列出不支援功能、重繪風險、撮合差異和多週期提醒。

04

執行門禁

分別給出可回測與可優化狀態；證據不足時 fail closed。

RESEARCH REPORTS

不要只看一條權益曲線。

繼續查看分布、商品明細、參數依據與穩定性邊界，區分普遍有效、少數商品貢獻和過擬合。

  • 組合權益與回撤
  • 正收益商品 / 總商品
  • 中位收益與中位回撤
  • 交易數與勝率
  • Sharpe 與盈利因子
  • 商品級權益曲線
Aniki 會依據現有指標給出簡短結論；模型不可用時仍會生成不捏造資料的計算結論。

FAQ

開始之前，你可能想知道。

01不會寫程式也能建立策略嗎？＋

可以。Aniki 會透過對話追問並整理成策略條件。

02Pine Script 是直接執行嗎？＋

不是。當前僅支援 Pine v5/v6 的安全子集，並先經語義、來源與執行門禁；不等同完整 TradingView 相容。

03參數優化只是暴力搜尋嗎？＋

產品先識別參數和研究價值，再生成受約束粗掃並檢查結果有效性。

04會連接交易所自動下單嗎？＋

不會。當前是本地策略研究工具，沒有實盤下單連接。
